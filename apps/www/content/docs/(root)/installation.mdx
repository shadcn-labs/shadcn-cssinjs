title: Installation
description: Set up StyleX and Base UI, then install components from the shadcn-cssinjs registry.
---

These components are styled with StyleX and built on Base UI. The
`shadcn-cssinjs` CLI configures the StyleX build pipeline and then installs the
same source-owned component files you would expect from shadcn/ui.

<Callout>
  Already using shadcn/ui? You can drop these components alongside your existing
  ones — they reuse the same CSS variables (`--background`, `--primary`, …) for
  theming.
</Callout>

## Initialize your project

<Steps>

<Step>Run the CLI</Step>

```bash
pnpm dlx shadcn-cssinjs@latest init
```

The CLI detects Next.js or Vite, installs the current official StyleX packages,
configures the appropriate bundler plugin, creates the import alias when needed,
adds the theme variables, and writes `components.json`.

For a new application, pass a directory name and choose a template:

```bash
pnpm dlx shadcn-cssinjs@latest init --name my-app --template next
```

Use `--template vite` for a Vite React app. Existing applications are detected
automatically, so the template flag is not required there.

<Step>Add a component</Step>

```bash
pnpm dlx shadcn-cssinjs@latest add button
```

The shared StyleX tokens are installed automatically the first time you
initialize or add a component.

</Steps>

## Install a component

```bash
pnpm dlx shadcn-cssinjs@latest add button
```

The component is copied into `components/ui/<name>/`. Import and use it:

```tsx
import { Button } from "@/components/ui/button/button";

export function Demo() {
  return <Button>Click me</Button>;
}
```

Browse the [components](/docs/components) for the full list and per-component install
commands.
